# GT911 Register Notes

Goodix GT911 — capacitive touch controller.
Local datasheet: `gt911.pdf`.

The public copies of this datasheet vary in quality. The driver currently uses
the stable host-facing I2C register surface: product ID, touch status, first
point data, and status clear.

## Interface

- Bus: I2C
- Default 7-bit address: `0x5d`
- Alternate 7-bit address: `0x14`
- Register address width: 16-bit big-endian
- Data byte order: little-endian for coordinate fields

## Register Summary

| Addr | Name | Driver Enum | Purpose |
|------|------|-------------|---------|
| 0x8040 | COMMAND | `command` | Controller command register |
| 0x8047 | CONFIG | `config` | Start of runtime configuration block |
| 0x8140 | PRODUCT_ID | `product_id` | ASCII product identifier |
| 0x814E | STATUS | `status` | Touch data-ready flag and point count |
| 0x814F | POINT1 | `point1` | First touch point record |

## Touch Status

`STATUS` is polled before reading point data.

| Bits | Meaning |
|------|---------|
| 7 | Data ready when set |
| 3:0 | Number of active touch points |

After the host reads touch data, it must write `0x00` back to `STATUS` so the
controller can publish the next report.

## Point Format

The current driver treats every touch point as an 8-byte record.

| Offset | Field | Format |
|--------|-------|--------|
| 0 | Track ID | u8 |
| 1..2 | X coordinate | little-endian u16 |
| 3..4 | Y coordinate | little-endian u16 |
| 5..6 | Touch size / pressure | little-endian u16 |
| 7 | Reserved | u8 |

## Driver Notes

- `open()` verifies the controller by reading `PRODUCT_ID`.
- `read()` returns zero when the ready bit is clear or the point count is zero.
- The driver clears `STATUS` on all read paths after observing a report.
- Coordinate orientation is board policy, so `Transform` handles swap and axis
  inversion outside the chip register logic.
