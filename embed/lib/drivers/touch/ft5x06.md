# FT5x06 Register Map

FocalTech FT5x06 series true multi-touch capacitive touch panel controller.
Datasheet: FT5x06, Document Number D-FT5x06-1212-V4.0, Version 4.0.

I2C address: default 0x38.

## Supported Parts

The FT5x06 series includes FT5206, FT5306, FT5406 and compatible FocalTech
touch controllers. The driver currently exposes the common 5-point operating
mode register map used by FT5206-style controllers.

## Touch Point Encoding

`TD_STATUS[3:0]` reports the number of active touch points.

Each point starts at `TOUCHn_XH` and uses four data bytes:

| Byte | Bits | Description |
|------|------|-------------|
| XH | 7:6 | Event flag: 0 put down, 1 put up, 2 contact, 3 reserved |
| XH | 3:0 | X position high nibble |
| XL | 7:0 | X position low byte |
| YH | 7:4 | Touch ID |
| YH | 3:0 | Y position high nibble |
| YL | 7:0 | Y position low byte |

Point register groups are spaced by 6 bytes: 0x03, 0x09, 0x0F, 0x15, 0x1B.

## Driver Usage

`open()` writes any non-null fields from `Parameters`.

The SZP board currently uses:

| Parameter | Register | Value |
|-----------|----------|-------|
| `valid_touch_threshold` | 0x80 | 70 |
| `peak_detect_threshold` | 0x81 | 60 |
| `focus_threshold` | 0x82 | 16 |
| `water_threshold` | 0x83 | 60 |
| `temperature_threshold` | 0x84 | 10 |
| `touch_difference_threshold` | 0x85 | 20 |
| `monitor_enter_time` | 0x87 | 2 |
| `active_period` | 0x88 | 12 |
| `monitor_period` | 0x89 | 40 |

The driver can apply a board-level coordinate transform after decoding raw
points. This is useful when the LCD panel rotation differs from the touch
controller's raw coordinate space.
