# ST7701 Register Notes

Sitronix ST7701S — TFT LCD controller/driver for RGB/MIPI-style panels.
Local datasheet: `st7701.pdf`.

The current Waveshare P4 4.3 board uses the ESP-IDF `esp_lcd_st7701` component
with MIPI DSI DBI commands and DPI pixel output. A generic driver should model
the controller command/init surface while leaving DSI bus, DPI timing, LDO, and
framebuffer policy in the ESP board layer.

## Interface

- Current board path: MIPI DSI command interface plus DPI video output
- Pixel format used by current board path: RGB565
- Panel timing is board data, not controller-global data

## Command Summary

| Cmd | Name | Purpose |
|-----|------|---------|
| 0x01 | SWRESET | Software reset |
| 0x10 | SLPIN | Enter sleep mode |
| 0x11 | SLPOUT | Exit sleep mode |
| 0x28 | DISPOFF | Display off |
| 0x29 | DISPON | Display on |
| 0x2A | CASET | Column address set |
| 0x2B | RASET | Row address set |
| 0x2C | RAMWR | Memory write |
| 0x36 | MADCTL | Memory access control |
| 0x3A | COLMOD | Pixel format |
| 0xFF | CMD2BKxSEL | Vendor command-bank selection |

## Current Board Init Shape

The existing P4 board-specific C path configures:

- MIPI DSI bus and DBI IO
- DPI panel timing
- ST7701 vendor init command list
- RGB565 input/output format
- DMA2D enablement
- reset GPIO, backlight PWM, and DSI PHY LDO

## Driver Notes

- Vendor command-bank sequences should be represented as init command data.
- MIPI DSI bus setup, DPI timing, number of framebuffers, and DMA2D are ESP
  platform concerns.
- `drawRgb565()` should remain rectangle-based. Board/platform code is
  responsible for splitting transfers if its DMA engine cannot accept the full
  region.
