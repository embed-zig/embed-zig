# SH8601 Register Notes

Shenghe SH8601A — 480x480 AMOLED display driver.
Local datasheet: `sh8601.pdf`.

The current Waveshare AMOLED board uses the ESP-IDF `esp_lcd_sh8601` component
over QSPI. A generic driver should keep controller commands and panel init data
separate from ESP bus setup and board power sequencing.

## Interface

- Common host interface in this repo: QSPI panel IO through ESP-IDF `esp_lcd`
- Native target class: 480RGB x 480 AMOLED panels
- Pixel format used by current board path: RGB565

## Command Summary

| Cmd | Name | Purpose |
|-----|------|---------|
| 0x11 | SLPOUT | Exit sleep mode |
| 0x28 | DISPOFF | Display off |
| 0x29 | DISPON | Display on |
| 0x2A | CASET | Column address set |
| 0x2B | RASET | Row address set |
| 0x2C | RAMWR | Memory write |
| 0x35 | TEON | Tearing effect line enable |
| 0x36 | MADCTL | Memory access control |
| 0x3A | COLMOD | Pixel format |
| 0x51 | WRDISBV | Display brightness |
| 0x53 | WRCTRLD | Brightness/control display |

## Current Board Init Shape

The existing board-specific C path configures:

- QSPI bus and panel IO
- controller init command list
- RGB element order
- 16-bit pixel mode
- optional transfer-complete semaphore for flush pacing

## Driver Notes

- Keep SH8601 command data in `embed`; keep SPI/QSPI bus creation in `esp`.
- Brightness writes should be a driver operation, but AMOLED power tree and
  reset sequencing should remain board-owned.
- The generic flush API should accept a full rectangle and not expose DMA row
  chunking to zux/LVGL code.
