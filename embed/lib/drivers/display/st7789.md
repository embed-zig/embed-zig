# ST7789 Register Notes

Sitronix ST7789V — single-chip TFT LCD controller/driver.
Local datasheet: `st7789.pdf`.

This part is commonly used behind SPI-connected 240x240, 240x320, and 320x240
RGB565 panels. The board layer owns reset, chip-select, data/command, backlight,
and panel-specific orientation.

## Interface

- Common host interface in this repo: SPI panel IO through ESP-IDF `esp_lcd`
- Pixel format used by current boards: RGB565, 16 bits per pixel
- Addressing model: column/page window followed by memory write

## Command Summary

| Cmd | Name | Purpose |
|-----|------|---------|
| 0x01 | SWRESET | Software reset |
| 0x11 | SLPOUT | Exit sleep mode |
| 0x13 | NORON | Normal display mode |
| 0x20 | INVOFF | Disable display inversion |
| 0x21 | INVON | Enable display inversion |
| 0x28 | DISPOFF | Display off |
| 0x29 | DISPON | Display on |
| 0x2A | CASET | Column address set |
| 0x2B | RASET | Row address set |
| 0x2C | RAMWR | Memory write |
| 0x36 | MADCTL | Memory access control |
| 0x3A | COLMOD | Pixel format |

## Driver Notes

- A generic driver should expose init commands as data, not hard-code one board
  sequence.
- Orientation should map to `MADCTL` plus optional board-level coordinate
  transform.
- `drawRgb565()` should set `CASET` and `RASET`, then write the RGB565 payload
  with `RAMWR`.
- DMA transfer limits are platform policy. The display driver can accept a full
  rectangle and let the ESP board implementation segment transfers as needed.
